Inventory Schema

Specification

Inventory:
  description: Inventory represents a restaurant inventory.
  properties:
    item-name: 
      type: text 
      column-name: Name 
      description: Name of the inventory item.  Format does not matter. 
    type: 
      type: one-of mass or volume
      column-name: Type 
      description: Defines the inventory item's measurement in volume or mass.

Recipe Schema

Specification

Recipes:
  description: Recipes represents the inventory items and thier amounts required to create a product.
  properties:
    recipe-name:
      type: text 
      column-name: Name 
      description: Name of the recipe. Format does not matter.
    ingredient-type:
      type: one-of "list" or "alternative"
      column-name: IngredientType
      description: Describes if the ingredients change to an "alternative" if ingredients are missing.
    ingredients:
      type: list
      column-name: all-of IngredientName, Amount, Unit
      description: a list of ingredients or alternatives
      properties:
        ingredient-name:
          type: one-of recipe-name or inventory-item-name
          column-name: IngredientName 
          description: Name of ingredient that must be a recipe name or an inventory item name.
        amount: 
          type: number 
          column-name: Amount 
          description: Amount of the ingredient that contributes to the recipe.
        unit: 
          type: one-of symbol of the metric or imperial system
          column-name: Unit 
          description: The unit of measurement for the ingredient.

Product Schema

Specification

Products:
  description: Products describe the recipe composition and their prices to create a sellable product.
  properties:
    product-name:
      type: text 
      column-name: Name 
      description: Name of the product being sold. Format does not matter.
    item:
      type: list
      column-name: all-of ItemName, IsRequired, Choices, Upcharge
      description: List of component/s and/or sides of a product.
      properties:
        item-name: 
          type: text 
          column-name: ItemName 
          description: Name of the item of a product.
        is-required: 
          type: boolean (TRUE or FALSE)
          column-name: IsRequired 
          description: Determines if the item is required.
        choices: 
          type: list 
          column-name: all-of Choices
          description: List of choices for a particular product item. 
          properties: 
            choice-name: 
              type: one-of a Products or Recipes name. 
              column-name: Choices 
              description: Name of the item choice. Must be a the name of another Products or a Recipes.

Sales Data

Specification

Sales-Data:
  description: Sales of a particular time period.
  properties:
    product-name: 
      type: product-name
      column-name: Name 
      description: Name of the product sold.
    amount: 
      type: number 
      column-name: Amount 
      description: Total number of Products sold in a time period.
    price:
      type: number 
      column-name: Price
      description: Price of the product at time-of-sale.    
    date-sold:
      type: date 
      column-name: Date 
      description: Date when the product was sold. 
    choices: 
      type: list 
      column-name: all-of Choices, ChoiceAmount 
      description: Total number of each possible choice sold in the date-sold time period. 
      properties: 
        choice-name: 
          type: Products choices choice-name 
          column-name: Choices 
          description: The name of a particular choice for a product.
        upcharge:
          type: number 
          column-name: Upcharge 
          description: Additional charge for a particular Choice at time-of-sale.  
        choice-amount: 
          type: number 
          column-name: ChoiceAmount 
          description: Total number of a particular choice for the given date-sold time period.
            The sum of all ChoiceAmounts must equal to the Sales amount.
